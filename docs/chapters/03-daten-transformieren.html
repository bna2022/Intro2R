4.1 Kontrollstrukturen

Mit Kontrollstrukturen kann man:

  • mehrere Befehle zusammenfassen,
  • Entscheidungen treffen,
  • Abläufe wiederholen.

4.1.1 Anweisungsblöcke

Ein Anweisungsblock fasst mehrere Befehle zusammen. In R werden Blöcke mit geschweiften Klammern { } definiert.

4.1.1.1 Syntax

{
  Befehl1
  Befehl2
  Befehl3
}

Beispiel: Berechnung eines BMI

{
  gewicht <- 72
  groesse <- 1.78
  bmi <- gewicht / groesse^2
  bmi
}
[1] 22.7244

Alle Befehle werden nacheinander ausgeführt. Der letzte Ausdruck wird ausgegeben.

4.1.2 Bedingte Anweisungen und Verzweigungen

Mit Bedingungen kann Code nur dann ausgeführt werden, wenn bestimmte Kriterien erfüllt sind.

4.1.2.1 if-Anweisung

Syntax

if (Bedingung) {
  Befehle
}

Beispiel: Fieber erkennen

temperatur <- 38.5

if (temperatur >= 38) {
  print("Fieber vorhanden")
}
[1] "Fieber vorhanden"

4.1.2.2 if … else

Syntax

if (Bedingung) {
  Befehle_wenn_wahr
} else {
  Befehle_wenn_falsch
}

Beispiel: Blutdruckbewertung

systole <- 128

if (systole >= 140) {
  print("Hypertonie möglich")
} else {
  print("Normalbereich")
}
[1] "Normalbereich"

4.1.2.3 if … else if … else

Mit dieser Struktur können mehrere Fälle unterschieden werden.

Beispiel: BMI-Klassifikation

bmi <- 27.4

if (bmi < 18.5) {
  print("Untergewicht")
} else if (bmi < 25) {
  print("Normalgewicht")
} else if (bmi < 30) {
  print("Übergewicht")
} else {
  print("Adipositas")
}
[1] "Übergewicht"

4.1.2.4 Wichtige Vergleichsoperatoren

Operator Bedeutung
== gleich
!= ungleich
> größer
< kleiner
>= größer gleich
<= kleiner gleich

4.1.2.5 Logische Operatoren

Operator Bedeutung
& und
| oder
! nicht

Beispiel

alter <- 67
raucher <- TRUE

if (alter > 65 & raucher) {
  print("Erhöhtes Risiko")
}
[1] "Erhöhtes Risiko"

4.1.3 Schleifen

Schleifen wiederholen Befehle mehrfach.

4.1.3.1 for-Schleife

Die for-Schleife wird verwendet, wenn bekannt ist, wie oft etwas wiederholt werden soll.

Syntax

for (variable in werte) {
  Befehle
}

Beispiel: Messwerte ausgeben

werte <- c(4.2, 5.1, 6.0)

for (wert in werte) {
  print(wert)
}
[1] 4.2
[1] 5.1
[1] 6

Beispiel: Mittelwert mehrerer Proben berechnen

proben <- c(8.1, 7.9, 8.4, 8.0)

summe <- 0

for (wert in proben) {
  summe <- summe + wert
}

mittelwert <- summe / length(proben)
mittelwert
[1] 8.1

4.1.3.2 while-Schleife

Die while-Schleife läuft so lange, wie eine Bedingung wahr ist.

Syntax

while (Bedingung) {
  Befehle
}

Beispiel: Zellwachstum

zellen <- 100

while (zellen < 1000) {
  zellen <- zellen * 2
  print(zellen)
}
[1] 200
[1] 400
[1] 800
[1] 1600

4.1.3.3 repeat-Schleife

Die repeat-Schleife läuft unbegrenzt, bis sie mit break beendet wird.

Beispiel

wert <- 1

repeat {
  wert <- wert * 2
  print(wert)

  if (wert > 100) {
    break
  }
}
[1] 2
[1] 4
[1] 8
[1] 16
[1] 32
[1] 64
[1] 128

4.1.3.4 break und next

4.1.3.4.1 break

break beendet eine Schleife sofort.

for (i in 1:10) {
  if (i == 5) {
    break
  }
  print(i)
}
[1] 1
[1] 2
[1] 3
[1] 4
4.1.3.4.2 next

next überspringt den aktuellen Schleifendurchlauf.

for (i in 1:5) {
  if (i == 3) {
    next
  }
  print(i)
}
[1] 1
[1] 2
[1] 4
[1] 5

4.1.4 Typische Fehler

4.1.4.1 Fehlende Klammern

x <- 5
if (x > 5)
  print(x)

Besser:

x <- 6
if (x > 5) {
  print(x)
}
[1] 6

4.1.4.2 Endlosschleife

while (x < 10) {
  print(x)
}

Hier wird x nie verändert. Dadurch endet die Schleife nicht.

4.1.5 Zusammenfassung

Struktur Zweck
{} Mehrere Befehle zusammenfassen
if Bedingte Ausführung
if ... else Zwei Alternativen
if ... else if Mehrere Fälle
for Wiederholung mit bekannter Anzahl
while Wiederholung mit Bedingung
repeat Wiederholung bis break

5 apply-Funktionen

Die apply-Familie besteht aus mehreren Funktionen, mit denen man eine Funktion wiederholt auf mehrere Elemente eines Objekts anwenden kann.

Sie wird häufig verwendet, wenn man sonst eine Schleife schreiben würde.

Typische Anwendungen sind:

  • Mittelwerte mehrerer Spalten berechnen
  • Funktionen auf jede Spalte eines Data Frames anwenden
  • Gruppenweise Kennwerte berechnen
  • Mehrere Vektoren gleichzeitig verarbeiten
  • Datentypen prüfen
  • fehlende Werte zählen

Die wichtigsten Funktionen sind:

Funktion Typischer Einsatz Ergebnis
apply() Matrizen, Zeilen oder Spalten Vektor, Matrix oder Liste
lapply() Listen oder Data Frames immer Liste
sapply() Listen oder Data Frames vereinfachtes Ergebnis
tapply() Werte nach Gruppen auswerten gruppiertes Ergebnis
mapply() mehrere Vektoren gleichzeitig Vektor, Matrix oder Liste

5.1 Grundidee der apply-Funktionen

Viele Aufgaben folgen demselben Muster:

Nimm mehrere Elemente und wende auf jedes Element dieselbe Funktion an.

Zum Beispiel:

x <- list(
  a = c(1, 2, 3),
  b = c(4, 5, 6),
  c = c(7, 8, 9)
)

Wir möchten für jedes Element den Mittelwert berechnen.

Mit einer Schleife:

ergebnis <- list()

for (i in names(x)) {
  ergebnis[[i]] <- mean(x[[i]])
}

ergebnis

Mit lapply() kürzer:

lapply(x, mean)

Die apply-Funktion übernimmt hier die Wiederholung.

5.2 apply()

apply() wird vor allem bei Matrizen verwendet.

Allgemeine Form:

apply(X, MARGIN, FUN)

Dabei bedeutet:

Argument Bedeutung
X Objekt, meist Matrix
MARGIN = 1 Funktion zeilenweise anwenden
MARGIN = 2 Funktion spaltenweise anwenden
FUN Funktion, die angewendet werden soll

5.2.1 Beispielmatrix

werte <- matrix(
  c(5, 6, 7,
    8, 9, 10,
    11, 12, 13),
  nrow = 3,
  byrow = TRUE
)

werte

Die Matrix sieht so aus:

     [,1] [,2] [,3]
[1,]    5    6    7
[2,]    8    9   10
[3,]   11   12   13

5.2.2 apply() zeilenweise

Mit MARGIN = 1 wird die Funktion auf jede Zeile angewendet.

apply(werte, 1, mean)

Ausgabe:

[1]  6  9 12

Erklärung:

mean(c(5, 6, 7))
mean(c(8, 9, 10))
mean(c(11, 12, 13))

R berechnet also den Mittelwert jeder Zeile.

5.2.3 apply() spaltenweise

Mit MARGIN = 2 wird die Funktion auf jede Spalte angewendet.

apply(werte, 2, mean)

Ausgabe:

[1]  8  9 10

Erklärung:

mean(c(5, 8, 11))
mean(c(6, 9, 12))
mean(c(7, 10, 13))

R berechnet also den Mittelwert jeder Spalte.

5.2.4 Weitere Beispiele mit apply()

Maximum pro Zeile:

apply(werte, 1, max)

Minimum pro Spalte:

apply(werte, 2, min)

Summe pro Zeile:

apply(werte, 1, sum)

Standardabweichung pro Spalte:

apply(werte, 2, sd)

5.2.5 apply() mit eigener Funktion

Man kann auch eigene Funktionen verwenden.

apply(werte, 1, function(x) max(x) - min(x))

Hier wird für jede Zeile die Spannweite berechnet.

Ausgabe:

[1] 2 2 2

Die anonyme Funktion

function(x) max(x) - min(x)

nimmt jeweils eine Zeile entgegen und berechnet:

größter Wert - kleinster Wert

5.2.6 Wichtiger Hinweis zu Data Frames

apply() kann auch auf Data Frames angewendet werden, ist dort aber mit Vorsicht zu verwenden.

Beispiel:

df <- data.frame(
  alter = c(23, 34, 45),
  groesse = c(1.70, 1.82, 1.65)
)

apply(df, 2, mean)

Das funktioniert, weil alle Spalten numerisch sind.

Problematisch wird es bei gemischten Datentypen:

df2 <- data.frame(
  name = c("Anna", "Ben", "Clara"),
  alter = c(23, 34, 45),
  groesse = c(1.70, 1.82, 1.65)
)

apply(df2, 2, mean)

Hier kann es zu Problemen kommen, weil apply() den Data Frame intern oft in eine Matrix umwandelt.

Eine Matrix kann aber nur einen Datentyp enthalten. Wenn eine Textspalte vorhanden ist, werden deshalb häufig auch die numerischen Werte in Text umgewandelt.

Für Data Frames sind deshalb oft lapply() oder sapply() besser geeignet.

5.3 lapply()

lapply() steht für list apply.

Allgemeine Form:

lapply(X, FUN)

Wichtig:

lapply() gibt immer eine Liste zurück.

5.3.1 Beispiel mit Liste

messwerte <- list(
  patient_a = c(120, 125, 130),
  patient_b = c(140, 138, 142),
  patient_c = c(110, 115, 118)
)

lapply(messwerte, mean)

Ausgabe:

$patient_a
[1] 125

$patient_b
[1] 140

$patient_c
[1] 114.3333

Jedes Listenelement wird einzeln an mean() übergeben.

5.3.2 lapply() bei Data Frames

Ein Data Frame ist intern eine Liste von Spalten.

Deshalb kann man lapply() sehr gut auf Data Frames anwenden.

daten <- data.frame(
  alter = c(23, 34, 45),
  gewicht = c(65, 82, 74),
  puls = c(70, 80, 76)
)

lapply(daten, mean)

Ausgabe:

$alter
[1] 34

$gewicht
[1] 73.66667

$puls
[1] 75.33333

Hier wird der Mittelwert für jede Spalte berechnet.

5.3.3 Datentypen mit lapply() prüfen

lapply(daten, class)

Ausgabe:

$alter
[1] "numeric"

$gewicht
[1] "numeric"

$puls
[1] "numeric"

Das ist besonders hilfreich, wenn man einen Data Frame untersucht.

5.3.4 lapply() mit eigener Funktion

lapply(daten, function(x) {
  c(
    mittelwert = mean(x),
    minimum = min(x),
    maximum = max(x)
  )
})

Ausgabe:

$alter
mittelwert   minimum   maximum 
       34        23        45 

$gewicht
mittelwert   minimum   maximum 
 73.66667        65        82 

$puls
mittelwert   minimum   maximum 
 75.33333        70        80

Hier wird auf jede Spalte dieselbe kleine Auswertungsfunktion angewendet.

5.4 sapply()

sapply() ist ähnlich wie lapply().

Der wichtigste Unterschied:

sapply() versucht, das Ergebnis zu vereinfachen.

Das bedeutet:

  • Wenn möglich, wird eine Liste in einen Vektor umgewandelt.
  • Wenn möglich, wird eine Liste in eine Matrix umgewandelt.
  • Wenn das nicht eindeutig möglich ist, bleibt das Ergebnis eine Liste.

5.4.1 Beispiel: lapply() versus sapply()

werte <- list(
  a = c(1, 2, 3),
  b = c(4, 5, 6)
)

lapply(werte, mean)

Ausgabe:

$a
[1] 2

$b
[1] 5

Mit sapply():

sapply(werte, mean)

Ausgabe:

a b 
2 5

Das Ergebnis ist jetzt ein Vektor.

5.4.2 sapply() bei Data Frames

sapply(daten, mean)

Ausgabe:

   alter  gewicht      puls 
34.00000 73.66667 75.33333

Das ist oft praktischer als lapply(), wenn man einen übersichtlichen numerischen Ergebnisvektor möchte.

5.4.3 sapply() zum Prüfen von Datentypen

patienten <- data.frame(
  id = c("P001", "P002", "P003"),
  alter = c(45, 52, 39),
  gruppe = c("Kontrolle", "Therapie", "Therapie"),
  crp = c(3.2, 7.8, 5.1)
)

sapply(patienten, class)

Mögliche Ausgabe:

         id       alter      gruppe         crp 
"character"   "numeric" "character"   "numeric"

Das ist eine sehr typische Anwendung von sapply().

5.4.4 sapply() mit logischer Prüfung

Man kann mit sapply() prüfen, welche Spalten numerisch sind.

sapply(patienten, is.numeric)

Ausgabe:

    id  alter gruppe    crp 
 FALSE   TRUE  FALSE   TRUE

Damit sieht man sofort, auf welche Spalten man mathematische Funktionen anwenden kann.

5.4.5 Mittelwerte nur für numerische Spalten berechnen

numerisch <- sapply(patienten, is.numeric)

sapply(patienten[numerisch], mean)

Ausgabe:

alter   crp 
45.33  5.37

Das ist sehr praktisch bei Data Frames mit gemischten Datentypen.

5.5 tapply()

tapply() wird verwendet, wenn Werte nach Gruppen ausgewertet werden sollen.

Allgemeine Form:

tapply(X, INDEX, FUN)

Dabei bedeutet:

Argument Bedeutung
X numerischer Vektor mit Werten
INDEX Gruppierungsvariable
FUN Funktion, die pro Gruppe angewendet wird

5.5.1 Beispiel: Mittelwert nach Gruppe

crp <- c(3.2, 4.1, 7.8, 6.5, 5.9, 2.8)
gruppe <- c("Kontrolle", "Kontrolle", "Therapie",
            "Therapie", "Therapie", "Kontrolle")

tapply(crp, gruppe, mean)

Ausgabe:

Kontrolle  Therapie 
 3.366667  6.733333

Hier wird der Mittelwert von crp getrennt nach Gruppe berechnet.

5.5.2 tapply() mit anderen Funktionen

Maximum pro Gruppe:

tapply(crp, gruppe, max)

Minimum pro Gruppe:

tapply(crp, gruppe, min)

Median pro Gruppe:

tapply(crp, gruppe, median)

Standardabweichung pro Gruppe:

tapply(crp, gruppe, sd)

5.5.3 tapply() mit Data-Frame-Spalten

lab <- data.frame(
  patient_id = c("P001", "P002", "P003", "P004", "P005", "P006"),
  gruppe = c("Kontrolle", "Kontrolle", "Therapie",
             "Therapie", "Therapie", "Kontrolle"),
  alter = c(45, 52, 39, 61, 48, 55),
  crp_mg_l = c(3.2, 4.1, 7.8, 6.5, 5.9, 2.8)
)

tapply(lab$crp_mg_l, lab$gruppe, mean)

Ausgabe:

Kontrolle  Therapie 
 3.366667  6.733333

5.5.4 Mehrere Kennwerte mit tapply()

tapply() kann auch mit selbst geschriebenen Funktionen kombiniert werden.

tapply(lab$crp_mg_l, lab$gruppe, function(x) {
  c(
    mittelwert = mean(x),
    median = median(x),
    maximum = max(x)
  )
})

Das Ergebnis ist etwas komplexer, weil pro Gruppe mehrere Werte berechnet werden.

Eine übersichtlichere Variante ist:

aggregate(crp_mg_l ~ gruppe, data = lab, FUN = mean)

Oder für mehrere Kennwerte:

aggregate(crp_mg_l ~ gruppe, data = lab, FUN = function(x) {
  c(
    mittelwert = mean(x),
    median = median(x),
    maximum = max(x)
  )
})

Für den Einstieg ist tapply() aber sehr gut geeignet, um das Prinzip gruppierter Berechnungen zu verstehen.

5.6 mapply()

mapply() wird verwendet, wenn eine Funktion mehrere Eingaben gleichzeitig braucht.

Allgemeine Form:

mapply(FUN, ...)

Dabei steht ... für mehrere Vektoren oder Listen, die parallel verarbeitet werden.

5.6.1 Einfaches Beispiel

x <- c(1, 2, 3)
y <- c(10, 20, 30)

mapply(function(a, b) a + b, x, y)

Ausgabe:

[1] 11 22 33

Das entspricht:

1 + 10
2 + 20
3 + 30

5.6.2 Beispiel: BMI berechnen

gewicht <- c(70, 82, 65)
groesse <- c(1.75, 1.82, 1.68)

mapply(function(g, h) g / h^2, gewicht, groesse)

Ausgabe:

[1] 22.85714 24.75546 23.03005

Hier bekommt die Funktion immer zwei Werte gleichzeitig:

gewicht[1] und groesse[1]
gewicht[2] und groesse[2]
gewicht[3] und groesse[3]

5.6.3 mapply() mit benannter Funktion

Man kann zuerst eine eigene Funktion schreiben:

bmi <- function(gewicht, groesse) {
  gewicht / groesse^2
}

Dann kann man diese Funktion mit mapply() verwenden:

mapply(bmi, gewicht, groesse)

Das ist oft übersichtlicher als eine anonyme Funktion.

5.7 apply-Funktionen im Vergleich

5.7.1 Beispiel-Datensatz

lab <- data.frame(
  patient_id = c("P001", "P002", "P003", "P004", "P005", "P006"),
  gruppe = c("Kontrolle", "Kontrolle", "Therapie",
             "Therapie", "Therapie", "Kontrolle"),
  alter = c(45, 52, 39, 61, 48, 55),
  crp_mg_l = c(3.2, 4.1, 7.8, 6.5, 5.9, 2.8),
  glucose = c(90, 95, 130, 125, 118, 88)
)

5.7.2 Welche Spalten sind numerisch?

sapply(lab, is.numeric)

Ausgabe:

patient_id     gruppe      alter   crp_mg_l    glucose 
     FALSE      FALSE       TRUE       TRUE       TRUE

5.7.3 Mittelwerte aller numerischen Spalten

num_spalten <- sapply(lab, is.numeric)

sapply(lab[num_spalten], mean)

Ausgabe:

    alter  crp_mg_l   glucose 
50.000000  5.050000 107.666667

5.7.4 Mittelwert von CRP nach Gruppe

tapply(lab$crp_mg_l, lab$gruppe, mean)

Ausgabe:

Kontrolle  Therapie 
 3.366667  6.733333

5.7.5 Maximum von Glucose nach Gruppe

tapply(lab$glucose, lab$gruppe, max)

Ausgabe:

Kontrolle  Therapie 
       95       130

5.8 Typische Fehler

5.8.1 Fehler 1: apply() auf gemischte Data Frames anwenden

apply(lab, 2, mean)

Problem:

Der Data Frame enthält Textspalten wie patient_id und gruppe. Dadurch kann mean() nicht sinnvoll auf alle Spalten angewendet werden.

Besser:

num_spalten <- sapply(lab, is.numeric)

sapply(lab[num_spalten], mean)

5.8.2 Fehler 2: lapply() gibt eine Liste zurück

lapply(lab[num_spalten], mean)

Das Ergebnis ist eine Liste.

Wenn man lieber einen Vektor möchte:

sapply(lab[num_spalten], mean)

5.8.3 Fehler 3: MARGIN bei apply() verwechseln

apply(werte, 1, mean)

bedeutet zeilenweise.

apply(werte, 2, mean)

bedeutet spaltenweise.

Merksatz:

1 steht für Zeilen, 2 steht für Spalten.

5.8.4 Fehler 4: Fehlende Werte nicht beachten

x <- c(3, 4, NA, 6)

mean(x)

Ausgabe:

[1] NA

Besser:

mean(x, na.rm = TRUE)

Das funktioniert auch mit apply-Funktionen:

sapply(lab[num_spalten], mean, na.rm = TRUE)

6 Funktionales Programmieren

Im vorherigen Abschnitt wurden die apply-Funktionen eingeführt. Dort stand vor allem die Frage im Mittelpunkt:

Wie kann eine Funktion auf mehrere Elemente angewendet werden?

Funktionales Programmieren geht einen Schritt weiter. Hier werden Funktionen selbst als Objekte behandelt. Das bedeutet:

  • Funktionen können gespeichert werden.
  • Funktionen können an andere Funktionen übergeben werden.
  • Funktionen können innerhalb anderer Funktionen erzeugt werden.
  • Funktionen können direkt dort definiert werden, wo sie gebraucht werden.

In R ist das besonders wichtig, weil viele Arbeitsschritte in der Datenanalyse immer wieder nach demselben Muster ablaufen:

Nimm Daten, wende eine Funktion darauf an und gib ein Ergebnis zurück.

6.1 Funktionen sind Objekte

In R kann eine Funktion wie ein anderes Objekt gespeichert werden.

quadrat <- function(x) {
  x^2
}

Hier wird eine Funktion erstellt und im Objekt quadrat gespeichert.

Die Funktion kann anschließend wie gewohnt aufgerufen werden:

quadrat(4)

Ausgabe:

[1] 16

Wichtig ist der Unterschied zwischen dem Funktionsobjekt und dem Funktionsaufruf.

quadrat

Ausgabe:

function(x) {
  x^2
}

Hier wird die Funktion selbst angezeigt.

quadrat(4)

Ausgabe:

[1] 16

Hier wird die Funktion ausgeführt.

Merksatz:

Ohne Klammern sieht man die Funktion.
Mit Klammern wird die Funktion ausgeführt.

6.2 Warum ist das nützlich?

Dass Funktionen als Objekte behandelt werden können, macht R sehr flexibel. Zum Beispiel können wir eine Funktion speichern:

kennzahl <- mean

Nun enthält kennzahl die Funktion mean().

kennzahl(c(2, 4, 6))

Ausgabe:

[1] 4

Das wirkt zunächst ungewohnt, zeigt aber eine wichtige Idee:

Funktionen können wie andere Objekte weitergegeben und verwendet werden.

Diese Idee steckt auch hinter vielen apply-Funktionen.

werte <- list(
  a = c(2, 4, 6),
  b = c(10, 20, 30)
)

sapply(werte, mean)

Ausgabe:

 a  b 
 4 20

Hier wird die Funktion mean an sapply() übergeben. sapply() entscheidet, auf welche Elemente mean() angewendet wird.

6.3 Anonyme Funktionen

Manchmal braucht man eine Funktion nur ein einziges Mal. Dann muss man ihr keinen eigenen Namen geben. Eine solche Funktion nennt man anonyme Funktion.

Beispiel:

sapply(1:5, function(x) x^2)

Ausgabe:

[1]  1  4  9 16 25

Die Funktion

function(x) x^2

wird direkt innerhalb von sapply() definiert. Sie hat keinen Namen, sondern wird sofort verwendet.

6.4 Anonyme Funktionen Schritt für Schritt

Der folgende Ausdruck kann zunächst schwer lesbar sein:

sapply(1:5, function(x) x^2)

Man kann ihn so lesen:

  1. Nimm die Werte 1:5.
  2. Wende auf jeden Wert eine Funktion an.
  3. Die Funktion heißt hier nicht extra.
  4. Für jeden Wert x wird x^2 berechnet.

Die anonyme Funktion ist also:

function(x) x^2

Sie entspricht inhaltlich dieser benannten Funktion:

quadrat <- function(x) {
  x^2
}

sapply(1:5, quadrat)

Ausgabe:

[1]  1  4  9 16 25

Beide Varianten liefern dasselbe Ergebnis.

Der Unterschied ist:

  • quadrat ist eine benannte Funktion.
  • function(x) x^2 ist eine anonyme Funktion.

6.5 Wann verwendet man anonyme Funktionen?

Anonyme Funktionen sind besonders praktisch, wenn eine Berechnung sehr kurz ist und nur an einer Stelle gebraucht wird.

Beispiel: Logarithmieren von Messwerten.

werte <- c(1, 10, 100)

sapply(werte, function(x) log10(x))

Ausgabe:

[1] 0 1 2

Hier wäre es möglich, vorher eine eigene Funktion zu definieren:

log_transform <- function(x) {
  log10(x)
}

sapply(werte, log_transform)

Ausgabe:

[1] 0 1 2

Für eine kurze einmalige Berechnung ist die anonyme Funktion aber kompakter.

6.6 Beispiel: Messwerte klassifizieren

Anonyme Funktionen können auch Bedingungen enthalten.

temperatur <- c(36.8, 37.6, 38.4, 39.1)

sapply(temperatur, function(x) {
  if (x < 37.5) {
    "normal"
  } else if (x < 38) {
    "erhöht"
  } else {
    "Fieber"
  }
})

Ausgabe:

[1] "normal" "erhöht" "Fieber" "Fieber"

Hier wird jede Temperatur einzeln bewertet. Die Funktion bekommt jeweils einen einzelnen Wert x und gibt eine Kategorie zurück.

6.7 Funktionen als Argumente

Eine Funktion kann an eine andere Funktion übergeben werden. Das wurde bei den apply-Funktionen bereits genutzt. Man kann dieses Prinzip auch in eigenen Funktionen verwenden.

analyse <- function(x, FUN) {
  FUN(x)
}

Diese Funktion hat zwei Argumente:

Argument Bedeutung
x Daten, die ausgewertet werden sollen
FUN Funktion, die auf x angewendet werden soll

Beispiel:

analyse(c(1, 2, 3), mean)

Ausgabe:

[1] 2

Nun mit einer anderen Funktion:

analyse(c(1, 2, 3), sd)

Ausgabe:

[1] 1

Die Funktion analyse() bleibt gleich. Nur die übergebene Funktion ändert sich.

6.8 Warum Funktionen als Argumente hilfreich sind

Das folgende Beispiel zeigt den Vorteil. Ohne funktionales Programmieren könnte man mehrere einzelne Funktionen schreiben:

analyse_mittelwert <- function(x) {
  mean(x)
}

analyse_standardabweichung <- function(x) {
  sd(x)
}

analyse_median <- function(x) {
  median(x)
}

Das ist möglich, aber wiederholt viel Struktur.

Mit einer Funktion, die eine andere Funktion als Argument erhält, wird der Code allgemeiner:

analyse <- function(x, FUN) {
  FUN(x)
}

Nun kann dieselbe Funktion unterschiedliche Kennzahlen berechnen:

werte <- c(4, 8, 10, 12)

analyse(werte, mean)
analyse(werte, median)
analyse(werte, sd)

Ausgabe:

[1] 8.5
[1] 9
[1] 3.41565

Die zentrale Idee lautet:

Nicht die Analysefunktion wird jedes Mal neu geschrieben, sondern die gewünschte Auswertungsfunktion wird übergeben.

6.9 Zusätzliche Argumente weitergeben

Viele Funktionen benötigen zusätzliche Argumente.

Beispiel:

werte <- c(4, 8, NA, 12)

mean(werte)

Ausgabe:

[1] NA

Mit na.rm = TRUE werden fehlende Werte ignoriert:

mean(werte, na.rm = TRUE)

Ausgabe:

[1] 8

Unsere Funktion analyse() kann so erweitert werden, dass zusätzliche Argumente weitergegeben werden.

analyse <- function(x, FUN, ...) {
  FUN(x, ...)
}

Die drei Punkte ... bedeuten:

Weitere Argumente werden an die innere Funktion weitergereicht.

Jetzt funktioniert:

analyse(werte, mean, na.rm = TRUE)

Ausgabe:

[1] 8

Das ist ein sehr häufiges Muster in R.

6.10 Funktionen erzeugen Funktionen

Funktionen können nicht nur Werte zurückgeben. Sie können auch neue Funktionen zurückgeben.

Beispiel:

multiplier <- function(faktor) {
  function(x) {
    x * faktor
  }
}

Diese Funktion erzeugt eine neue Funktion.

mal2 <- multiplier(2)
mal5 <- multiplier(5)

Nun sind mal2 und mal5 selbst Funktionen.

mal2(10)

Ausgabe:

[1] 20
mal5(10)

Ausgabe:

[1] 50

6.11 Funktionen erzeugen Funktionen: Schrittweise Erklärung

Die Funktion multiplier() bekommt einen Faktor.

multiplier <- function(faktor) {
  function(x) {
    x * faktor
  }
}

Wenn wir schreiben:

mal2 <- multiplier(2)

dann wird eine neue Funktion erzeugt.

Diese neue Funktion merkt sich den Faktor 2.

Sie verhält sich ungefähr so:

mal2 <- function(x) {
  x * 2
}

Entsprechend erzeugt

mal5 <- multiplier(5)

eine Funktion, die mit 5 multipliziert.

mal5 <- function(x) {
  x * 5
}

Der wichtige Gedanke ist:

Eine Funktion kann eine andere Funktion bauen.

6.12 Beispiel: Grenzwertprüfungen erzeugen

Ein praktisches Beispiel ist das Erzeugen von Prüf-Funktionen.

groesser_als <- function(grenze) {
  function(x) {
    x > grenze
  }
}

Nun können spezielle Prüf-Funktionen erzeugt werden.

groesser_als_5 <- groesser_als(5)
groesser_als_10 <- groesser_als(10)

Anwendung:

groesser_als_5(c(3, 6, 9))

Ausgabe:

[1] FALSE  TRUE  TRUE
groesser_als_10(c(3, 6, 12))

Ausgabe:

[1] FALSE FALSE  TRUE

Die Funktion groesser_als() erzeugt also Funktionen, die jeweils eine bestimmte Grenze prüfen.

6.13 Beispiel mit Patientendaten

patienten <- data.frame(
  patient_id = c("P001", "P002", "P003", "P004", "P005"),
  crp_mg_l = c(3.2, 6.1, 8.4, 2.9, 5.7)
)

Wir erzeugen eine Funktion, die prüft, ob ein Wert über einer bestimmten Grenze liegt.

ueber_grenze <- function(grenze) {
  function(x) {
    x > grenze
  }
}

Nun erstellen wir eine konkrete Prüf-Funktion.

crp_erhoeht <- ueber_grenze(5)

Anwendung auf die CRP-Werte:

crp_erhoeht(patienten$crp_mg_l)

Ausgabe:

[1] FALSE  TRUE  TRUE FALSE  TRUE

Das Ergebnis kann als neue Spalte gespeichert werden:

patienten$crp_erhoeht <- crp_erhoeht(patienten$crp_mg_l)

patienten

Ausgabe:

  patient_id crp_mg_l crp_erhoeht
1       P001      3.2       FALSE
2       P002      6.1        TRUE
3       P003      8.4        TRUE
4       P004      2.9       FALSE
5       P005      5.7        TRUE

6.14 Typische Fehler

6.14.1 Funktion nicht aufrufen

Ein häufiger Fehler ist, dass eine Funktion ohne Klammern geschrieben wird.

quadrat

Das zeigt nur die Funktion an.

Die Funktion wird dadurch nicht ausgeführt.

Richtig ist:

quadrat(4)

Ausgabe:

[1] 16

Merksatz:

Eine Funktion wird erst durch runde Klammern ausgeführt.

6.14.2 Argumente vergessen

Wenn eine Funktion ein Argument erwartet, muss dieses beim Aufruf angegeben werden.

quadrat <- function(x) {
  x^2
}

quadrat()

Das führt zu einem Fehler, weil x fehlt.

Richtig:

quadrat(5)

Ausgabe:

[1] 25

6.14.3 Funktion und Funktionsaufruf verwechseln

Bei sapply() und ähnlichen Funktionen wird häufig eine Funktion übergeben.

sapply(1:5, sqrt)

Ausgabe:

[1] 1.000000 1.414214 1.732051 2.000000 2.236068

Hier steht sqrt ohne Klammern.

Das ist richtig, weil sapply() die Funktion selbst benötigt.

Falsch wäre:

sapply(1:5, sqrt())

Hier würde sqrt() sofort aufgerufen werden, aber ohne Argument.

Merksatz:

Wenn eine Funktion übergeben wird, schreibt man meistens den Funktionsnamen ohne Klammern.

6.14.4 Rückgabetyp nicht beachten

Anonyme Funktionen können unterschiedliche Rückgabetypen erzeugen.

Beispiel:

werte <- c(1, 2, 3)

sapply(werte, function(x) {
  if (x > 1) {
    x * 10
  } else {
    "klein"
  }
})

Ausgabe:

[1] "klein" "20"    "30"

Da einmal Text zurückgegeben wird, wandelt R auch die Zahlen in Text um.

Besser ist es, innerhalb einer Funktion möglichst einheitliche Rückgabetypen zu verwenden.

Zum Beispiel nur Text:

sapply(werte, function(x) {
  if (x > 1) {
    "größer als 1"
  } else {
    "klein"
  }
})

Ausgabe:

[1] "klein"        "größer als 1" "größer als 1"

Oder nur Zahlen:

sapply(werte, function(x) {
  if (x > 1) {
    x * 10
  } else {
    NA
  }
})

Ausgabe:

[1] NA 20 30

6.16 Zusammenfassung

Funktionales Programmieren bedeutet in R, dass Funktionen selbst als Objekte behandelt werden.

Wichtige Punkte:

  • Funktionen können gespeichert werden.
  • Funktionen können ohne direkten Aufruf an andere Funktionen übergeben werden.
  • Anonyme Funktionen sind nützlich für kurze einmalige Berechnungen.
  • Eigene Funktionen können andere Funktionen als Argument erhalten.
  • Mit ... können zusätzliche Argumente weitergegeben werden.
  • Funktionen können neue Funktionen erzeugen.
  • Beim Übergeben einer Funktion schreibt man den Funktionsnamen meist ohne Klammern.

Merksatz:

Funktionales Programmieren bedeutet: Funktionen sind nicht nur Werkzeuge, sondern selbst Objekte, mit denen man arbeiten kann.
