Vektoren

Vektoren: die zentrale Struktur

Ein Vektor enthält mehrere Werte hintereinander.

x <- c(1.7, 3.8, 4.2, 5.7)
x

Wichtig:

  • atomare Vektoren enthalten Werte eines Datentyps
  • viele R-Funktionen arbeiten automatisch elementweise
  • Vektoren sind Grundlage für viele andere Datenstrukturen

Vektoren erzeugen

c(1, 1.5, 2)

double(5)

1:5

rep(3, 5)

seq(0, 0.6, by = 0.2)

Mini-Frage: Was ist der Unterschied zwischen rep() und seq()?

rep wiederholt Werte; seq erzeugt eine Zahlenfolge mit Start, Ende und Schrittweite.

  1. 2 4 6 8 10
  2. fünfmal den Wert 0
  3. die Zahlen von 0 bis 1 in Schritten von 0.25
  4. den Vektor "ja" "nein" "ja"
# Ihre Lösung

Indizierung: Zugriff auf Elemente

In R beginnt die Zählung bei 1.

x <- c(1.7, 3.8, 4.2, 5.7)

x[2]
x[2:3]
x[c(4, 4, 1)]
x[-c(1, 2)]

Merksatz:

  • x[i]: einzelnes Element
  • x[i:j]: zusammenhängender Bereich
  • x[c(...)]: beliebige Auswahl
  • x[-i]: Element ausschließen

Logische Indizierung

Ein logischer Vektor wirkt wie ein Filter.

x <- c(1.7, 0.5, -0.7, 0, 2.8, 0.2)

x[c(TRUE, TRUE, FALSE, FALSE, TRUE, TRUE)]

TRUE lässt ein Element passieren, FALSE entfernt es.

Filtern mit Bedingungen

x <- c(0.3, 0.1, 5.7, -1.0)

x > 0

x[x > 0]

Eine Bedingung erzeugt zuerst einen logischen Vektor.

Dieser logische Vektor wird dann als Index benutzt.

Positionen finden mit which()

Manchmal interessieren nicht die Werte, sondern die Positionen.

x <- c(0.3, 0.1, 5.7, -1.0)

which(x > 0)

which() gibt die Positionen zurück, an denen eine Bedingung TRUE ist.

Vektoren kombinieren

x <- c(0.3, 0.1, 5.7, -1.0)
y <- c(2, 3, 1, 5)

c(x, y)

c() steht für combine.

Skalare Operationen

R rechnet elementweise.

x <- c(1, 1.5, 2)

x + 2
x - 2
x * 2
x / 2

Jedes Element wird einzeln verarbeitet.

Vektor-Arithmetik

Bei gleich langen Vektoren werden Elemente paarweise verrechnet.

c(1, 2, 3) + c(1.2, -0.5, -0.1)

c(1, 2, 3) * c(1.2, -0.5, -0.1)

Recycling-Prinzip

Wenn Vektoren unterschiedlich lang sind, wiederholt R den kürzeren Vektor.

c(1, 1, 1, 1, 1) * c(1, 2, 3)

R interpretiert dies intern ungefähr als:

c(1, 1, 1, 1, 1) * c(1, 2, 3, 1, 2)

Achten Sie auf Warnungen, wenn die Längen nicht gut zusammenpassen.

Funktionen auf Vektoren

Viele Funktionen arbeiten elementweise:

exp(c(0, 1, 2, 3))
sqrt(c(4, 9, 16))

Andere Funktionen fassen einen Vektor zu einem Wert zusammen:

mean(c(1.2, -0.5, -0.1))
sum(c(1, 2, 3))
length(c(1, 2, 3))

Skalarprodukt

a <- c(1, 2, 3)
b <- c(1.2, -0.5, -0.1)

a %*% b

Das Skalarprodukt multipliziert paarweise und summiert anschließend.

Aktivierung: CRP-Werte

crp <- c(2.1, 8.4, 0.6, 12.3, 3.2, 15.7, 1.0)

  1. Welche Werte sind größer als 5?
  2. Filtern Sie alle Werte über 5.
  3. Bestimmen Sie die Positionen dieser Werte.
  4. Ergänzen Sie crp_neu <- c(4.3, 9.8).

Matrizen

Was ist eine Matrix?

Eine Matrix ist ein zweidimensionaler Container.

Eigenschaften:

  • Zeilen × Spalten
  • alle Elemente haben denselben Datentyp
  • nützlich für numerische Berechnungen und lineare Algebra

Matrix erzeugen

x <- matrix(1:6, nrow = 2, ncol = 3)
x
     [,1] [,2] [,3]
[1,]    1    3    5
[2,]    2    4    6

Wichtig:

R füllt Matrizen standardmäßig spaltenweise.

Matrix-Indizierung

Allgemeine Form:

x[Zeile, Spalte]

Matrix-Indizierung

Beispiele:

x[1, 3]    # einzelnes Element
[1] 5
x[2, ]     # ganze Zeile
[1] 2 4 6
x[, 2]     # ganze Spalte
[1] 3 4
x[, 1:2]   # mehrere Spalten
     [,1] [,2]
[1,]    1    3
[2,]    2    4

drop = FALSE

x[2, , drop = FALSE]
     [,1] [,2] [,3]
[1,]    2    4    6
x[, 2, drop = FALSE]
     [,1]
[1,]    3
[2,]    4

Ohne drop = FALSE macht R aus einer einzelnen Zeile oder Spalte oft einen Vektor.

Mit drop = FALSE bleibt die Matrixstruktur erhalten.

Lineare Indizierung

Matrizen können auch wie ein Vektor indiziert werden.

x <- matrix(c(10, 50, 3, 99, -1, 8), nrow = 2)
x
     [,1] [,2] [,3]
[1,]   10    3   -1
[2,]   50   99    8
x[1]
[1] 10
x[5]
[1] -1

Bei x[…] ohne Komma wird keine Zeilen- und Spaltenstruktur mehr verwendet. R behandelt die Matrix dann wie einen langen Vektor. Deshalb nennt man das lineare Indizierung.

Lineare Indizierung

x[x < 3]
[1] -1

R prüft jedes Element der Matrix:

      [,1]  [,2]  [,3]
[1,] FALSE FALSE  TRUE
[2,] FALSE FALSE FALSE

Nur Elemente mit TRUE werden ausgewählt.

Das Ergebnis ist ein Vektor, keine Matrix.

Zeilen- und Spaltennamen

colnames(x) <- c("C1", "C2", "C3")
rownames(x) <- c("R1", "R2")

x
   C1 C2 C3
R1 10  3 -1
R2 50 99  8

Namen machen Code lesbarer.

Matrix-Funktionen (1/3)

Eine Matrix erzeugen:

     [,1] [,2] [,3]
[1,]    1    3    5
[2,]    2    4    6

Dimensionen abfragen:

dim(A)     # Zeilen und Spalten
[1] 2 3
nrow(A)   # Anzahl Zeilen
[1] 2
ncol(A)   # Anzahl Spalten
[1] 3

Matrix-Funktionen (2/3)

Anzahl Elemente abfragen:

length(A) # Anzahl gespeicherter Werte
[1] 6

Wichtig: length(A) zählt alle Elemente, nicht die Anzahl der Zeilen.

Matrix-Funktionen (3/3)

Transponieren einer Matrix:

A <- matrix(1:6, nrow = 2, ncol = 3)
A
     [,1] [,2] [,3]
[1,]    1    3    5
[2,]    2    4    6
t(A)
     [,1] [,2]
[1,]    1    2
[2,]    3    4
[3,]    5    6

t(A) vertauscht Zeilen und Spalten.

Matrixmultiplikation

Matrix A:

A <- matrix(1:6, nrow = 2, ncol = 3)
A
     [,1] [,2] [,3]
[1,]    1    3    5
[2,]    2    4    6

Matrix B:

B <- matrix(1:6, nrow = 3, ncol = 2)
B
     [,1] [,2]
[1,]    1    4
[2,]    2    5
[3,]    3    6
A %*% B
     [,1] [,2]
[1,]   22   49
[2,]   28   64

Wichtig: Für A %*% B muss die Anzahl der Spalten von A zur Anzahl der Zeilen von B passen.

Listen

Listen: flexible Container

Listen können verschiedene Objekte enthalten.

L <- list(servus="hallo", c(1, 3, 7), matrix(1:6, 3))
L
$servus
[1] "hallo"

[[2]]
[1] 1 3 7

[[3]]
     [,1] [,2]
[1,]    1    4
[2,]    2    5
[3,]    3    6

Eine Liste kann also Text, Vektoren, Matrizen oder weitere Listen enthalten.

Listen indizieren: [ ] vs. [[ ]]

Mit [ ] erhält man eine Teilliste:

L[1]
$servus
[1] "hallo"

Mit [[ ]] erhält man das gespeicherte Element selbst:

L[[1]]
[1] "hallo"

Merke:

  • L[i] gibt eine Liste zurück
  • L[[i]] gibt das Element zurück

Zugriff auf Listen

L$name greift auf ein benanntes Element zu

L$servus
[1] "hallo"

Listen: Zusammenfassung

Listen sind flexible Container für Objekte, die zusammengehören, aber unterschiedlich aufgebaut sein können.

Viele R-Objekte sind intern Listen, zum Beispiel Modelle oder Data Frames.

Data Frames

Data Frames

Ein Data Frame ist die zentrale Struktur für tabellarische Daten.

Eigenschaften:

  • zweidimensional: Zeilen und Spalten
  • jede Spalte ist ein Vektor
  • Spalten dürfen unterschiedliche Datentypen haben
  • innerhalb einer Spalte ist der Datentyp einheitlich

Data Frame erstellen

df <- data.frame(
  foo = 1:4,
  bar = c(TRUE, TRUE, FALSE, FALSE)
)

df
  foo   bar
1   1  TRUE
2   2  TRUE
3   3 FALSE
4   4 FALSE
nrow(df)
[1] 4
ncol(df)
[1] 2

Datensatz iris

data(iris)

101          6.3         3.3          6.0         2.5  virginica

iris ist ein eingebauter Beispieldatensatz.

! dreht logische Werte um:

  • aus TRUE wird FALSE
  • aus FALSE wird TRUE

Mehrere Werte mit %in%

   Sepal.Length Sepal.Width Petal.Length Petal.Width    Species
1           5.1         3.5          1.4         0.2     setosa

%in% prüft, ob ein Wert in einer Menge enthalten ist.

Sehr nützlich für mehrere Kategorien.

Fehlende Werte und complete.cases()

daten <- data.frame(
  id = 1:4,
  wert = c(2.3, NA, 4.1, 5.0)
)

complete.cases(daten)
[1]  TRUE FALSE  TRUE  TRUE
daten[complete.cases(daten), ]
  id wert
1  1  2.3
3  3  4.1

complete.cases() findet Zeilen ohne fehlende Werte.

sapply() auf Data Frames

sapply() wendet eine Funktion auf mehrere Elemente an.

Bei Data Frames typischerweise: auf jede Spalte.

daten <- data.frame(
  Alter = c(23, 31, 28),
  Note = c(1.7, 2.3, 1.9)
)

sapply(daten, mean)
    Alter      Note 
27.333333  1.966667 
sapply(daten, min)
Alter  Note 
 23.0   1.7

Spaltentypen prüfen

daten2 <- data.frame(
  Name = c("Anna", "Ben", "Clara"),
  Alter = c(23, 31, 28),
  Student = c(TRUE, FALSE, TRUE)
)

sapply(daten2, class)
       Name       Alter     Student 
"character"   "numeric"   "logical" 
sapply(daten2, is.numeric)
   Name   Alter Student 
  FALSE    TRUE   FALSE

sapply() hilft, Spalten systematisch zu untersuchen.

Häufigkeitstabellen mit table()

Mit table() kann man zählen, wie oft Werte vorkommen.

gruppe <- c("A", "A", "B", "B", "A", "B")
table(gruppe)
gruppe
A B 
3 3

table() erstellt eine absolute Häufigkeitstabelle.

table() mit Data Frames

In Data Frames nutzt man table() häufig für einzelne Spalten.

df <- data.frame(
  gruppe = c("A", "A", "B", "B", "A", "B"),
  bestanden = c("ja", "nein", "ja", "ja", "ja", "nein")
)

Eine Spalte auszählen:

table(df$gruppe)

A B 
3 3

Mit $ greifen wir auf eine einzelne Spalte des Data Frames zu.

Kreuztabellen

Mit table() können auch zwei Variablen gemeinsam ausgezählt werden.

table(df$gruppe, df$bestanden)
   
    ja nein
  A  2    1
  B  2    1

Etwas lesbarer mit Namen:

table(
  Gruppe = df$gruppe,
  Bestanden = df$bestanden
)
      Bestanden
Gruppe ja nein
     A  2    1
     B  2    1

Eine Kreuztabelle zeigt, wie häufig Kombinationen von Ausprägungen vorkommen.

Relative Häufigkeiten

Absolute Häufigkeiten zählen Fälle.

tab <- table(
  Gruppe = df$gruppe,
  Bestanden = df$bestanden
)
tab
      Bestanden
Gruppe ja nein
     A  2    1
     B  2    1

Relative Häufigkeiten zeigen Anteile.

prop.table(tab)
      Bestanden
Gruppe        ja      nein
     A 0.3333333 0.1666667
     B 0.3333333 0.1666667

prop.table() wandelt absolute Häufigkeiten in relative Häufigkeiten um.

Factors

Was sind Factors?

Factors repräsentieren kategoriale Variablen.

Beispiele:

  • Geschlecht
  • Farben
  • Gruppen
  • Bewertungen
  • Klassen

Die möglichen Kategorien heißen Levels.

Factor erstellen

farben <- factor(c("rot", "blau", "rot", "gruen", "blau"))

farben
[1] rot   blau  rot   gruen blau 
Levels: blau gruen rot
levels(farben)
[1] "blau"  "gruen" "rot"  
nlevels(farben)
[1] 3

R speichert Factors intern als Zahlen mit zugehörigen Levels.

Levels festlegen

farben <- factor(
  c("rot", "blau", "rot", "gruen"),
  levels = c("rot", "blau", "gruen")
)

farben

Die Reihenfolge der Levels kann für Plots und Modelle wichtig sein.

Geordnete Factors

bewertung <- factor(
  c("gut", "mittel", "gut", "schlecht"),
  levels = c("schlecht", "mittel", "gut"),
  ordered = TRUE
)

bewertung
[1] gut      mittel   gut      schlecht
Levels: schlecht < mittel < gut
is.ordered(bewertung)
[1] TRUE

Geordnete Factors besitzen eine natürliche Reihenfolge.

Zugriff auf Factor-Elemente

farben[1]
[1] rot
Levels: blau gruen rot
farben[1:3]
[1] rot  blau rot 
Levels: blau gruen rot

Factors verhalten sich beim Zugriff ähnlich wie Vektoren.

Typische Fehlerquelle: as.numeric()

x <- factor(c("10", "20", "30"))

as.numeric(x)
[1] 1 2 3

Das Ergebnis sind die internen Codes, nicht die sichtbaren Werte.

Richtig:

as.numeric(as.character(x))
[1] 10 20 30

tapply() mit Factors

tapply() berechnet Kennzahlen getrennt nach Gruppen.

umsatz <- c(100, 200, 150, 300, 250, 180)

abteilung <- factor(c(
  "Verkauf", "Verkauf", "IT",
  "IT", "Marketing", "Marketing"
))

tapply(umsatz, abteilung, sum)
       IT Marketing   Verkauf 
      450       430       300

Was passiert bei tapply()?

tapply(werte, gruppe, funktion)

  • werte: Werte, mit denen gerechnet wird
  • gruppe: Gruppierungsvariable, oft Factor
  • funktion: Funktion pro Gruppe

Beispiel:

tapply(umsatz, abteilung, mean)
tapply(umsatz, abteilung, max)

Leere Factor-Levels

abteilung <- factor(
  c("Verkauf", "Verkauf", "IT"),
  levels = c("IT", "Marketing", "Verkauf")
)

umsatz <- c(100, 200, 150)

tapply(umsatz, abteilung, sum)

Für Marketing gibt es keinen Wert, aber der Level existiert.

Deshalb erscheint NA.
