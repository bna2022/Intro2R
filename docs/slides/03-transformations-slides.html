Kontrollstrukturen

Was sind Kontrollstrukturen?

Kontrollstrukturen steuern, welcher Code ausgeführt wird.

Sie ermöglichen:

  • Entscheidungen
  • Wiederholungen
  • strukturierte Programmabläufe

Anweisungsblöcke

Mehrere Befehle können mit { } zusammengefasst werden.

{
  gewicht <- 72
  groesse <- 1.78
  bmi <- gewicht / groesse^2
  bmi
}
[1] 22.7244

Bedingte Ausführung mit if

Code wird nur ausgeführt, wenn eine Bedingung erfüllt ist.

temperatur <- 38.5

if (temperatur >= 38) {
  print("Fieber vorhanden")
}
[1] "Fieber vorhanden"

if und else

Mit else wird eine Alternative formuliert.

systole <- 128

if (systole >= 140) {
  print("Hypertonie möglich")
} else {
  print("Normalbereich")
}
[1] "Normalbereich"

Mehrere Fälle unterscheiden

Mit else if können mehrere Bedingungen geprüft werden.

bmi <- 27.4

if (bmi < 18.5) {
  print("Untergewicht")
} else if (bmi < 25) {
  print("Normalgewicht")
} else if (bmi < 30) {
  print("Übergewicht")
} else {
  print("Adipositas")
}
[1] "Übergewicht"

Vergleichs- und logische Operatoren

Operator Bedeutung
== gleich
!= ungleich
> größer
< kleiner
>= größer gleich
<= kleiner gleich
& und
| oder
! nicht

Beispiel: logische Verknüpfung

alter <- 67
raucher <- TRUE

if (alter > 65 & raucher) {
  print("Erhöhtes Risiko")
}
[1] "Erhöhtes Risiko"

Schleifen

Schleifen wiederholen Befehle mehrfach.

Besonders wichtig:

  • for: bekannte Anzahl von Wiederholungen
  • while: Wiederholung solange Bedingung erfüllt ist
  • repeat: Wiederholung bis break

for-Schleife

werte <- c(4.2, 5.1, 6.0)

for (wert in werte) {
  print(wert)
}
[1] 4.2
[1] 5.1
[1] 6

for-Schleife: Mittelwert

proben <- c(8.1, 7.9, 8.4, 8.0)

summe <- 0

for (wert in proben) {
  summe <- summe + wert
}

mittelwert <- summe / length(proben)
mittelwert
[1] 8.1

while-Schleife

Eine while-Schleife läuft, solange die Bedingung wahr ist.

zellen <- 100

while (zellen < 1000) {
  zellen <- zellen * 2
  print(zellen)
}
[1] 200
[1] 400
[1] 800
[1] 1600

break und next

  • break beendet eine Schleife
  • next überspringt den aktuellen Durchlauf
for (i in 1:5) {
  if (i == 3) {
    next
  }
  print(i)
}
[1] 1
[1] 2
[1] 4
[1] 5

Typische Fehler

  • fehlende { }
  • Bedingung ergibt nicht TRUE oder FALSE
  • Endlosschleife
  • Objekt wird innerhalb der Schleife nicht verändert

Übung: Kontrollstrukturen

apply-Funktionen

Grundidee

Viele Aufgaben folgen demselben Muster:

Wende dieselbe Funktion auf mehrere Elemente an.

Die apply-Familie ersetzt häufig Schleifen.

Typische Anwendungen:

  • Mittelwerte mehrerer Spalten berechnen
  • Datentypen prüfen
  • Gruppen vergleichen
  • fehlende Werte zählen

Überblick

Funktion Typischer Einsatz Ergebnis
apply() Matrizen Vektor, Matrix oder Liste
lapply() Listen, Data Frames immer Liste
sapply() Listen, Data Frames vereinfachtes Ergebnis
tapply() Gruppenvergleiche gruppiertes Ergebnis
mapply() mehrere Eingaben parallel Vektor, Matrix oder Liste

apply()

apply() wird häufig bei Matrizen verwendet.

Allgemeine Form:

apply(X, MARGIN, FUN)

Dabei gilt:

  • MARGIN = 1: zeilenweise
  • MARGIN = 2: spaltenweise
  • FUN: anzuwendende Funktion

Beispielmatrix

werte <- matrix(
  c(5, 6, 7,
    8, 9, 10,
    11, 12, 13),
  nrow = 3,
  byrow = TRUE
)

werte
     [,1] [,2] [,3]
[1,]    5    6    7
[2,]    8    9   10
[3,]   11   12   13

apply() zeilenweise

apply(werte, 1, mean)
[1]  6  9 12

MARGIN = 1 bedeutet:

Berechne den Mittelwert jeder Zeile.

apply() spaltenweise

apply(werte, 2, mean)
[1]  8  9 10

MARGIN = 2 bedeutet:

Berechne den Mittelwert jeder Spalte.

Weitere Beispiele

apply(werte, 1, max)
[1]  7 10 13
apply(werte, 2, min)
[1] 5 6 7
apply(werte, 1, sum)
[1] 18 27 36
apply(werte, 2, sd)
[1] 3 3 3

Eigene Funktion in apply()

apply(werte, 1, function(x) max(x) - min(x))
[1] 2 2 2

Hier wird für jede Zeile die Spannweite berechnet:

größter Wert - kleinster Wert

Vorsicht bei Data Frames

apply() wandelt Data Frames oft intern in Matrizen um.

Das ist problematisch, wenn verschiedene Datentypen enthalten sind.

df <- data.frame(
  name = c("Anna", "Ben", "Clara"),
  alter = c(23, 34, 45),
  groesse = c(1.70, 1.82, 1.65)
)

str(df)
'data.frame':   3 obs. of  3 variables:
 $ name   : chr  "Anna" "Ben" "Clara"
 $ alter  : num  23 34 45
 $ groesse: num  1.7 1.82 1.65

lapply()

lapply() steht für list apply.

Wichtig: lapply() gibt immer eine Liste zurück.

messwerte <- list(
  patient_a = c(120, 125, 130),
  patient_b = c(140, 138, 142),
  patient_c = c(110, 115, 118)
)

lapply(messwerte, mean)
$patient_a
[1] 125

$patient_b
[1] 140

$patient_c
[1] 114.3333

lapply() bei Data Frames

Ein Data Frame ist intern eine Liste von Spalten.

daten <- data.frame(
  alter = c(23, 34, 45),
  gewicht = c(65, 82, 74),
  puls = c(70, 80, 76)
)

lapply(daten, mean)
$alter
[1] 34

$gewicht
[1] 73.66667

$puls
[1] 75.33333

Datentypen prüfen

lapply(daten, class)
$alter
[1] "numeric"

$gewicht
[1] "numeric"

$puls
[1] "numeric"

Das ist besonders hilfreich, um einen Data Frame zu untersuchen.

sapply()

sapply() ist ähnlich wie lapply().

Der wichtigste Unterschied: sapply() versucht, das Ergebnis zu vereinfachen.

werte_liste <- list(
  a = c(1, 2, 3),
  b = c(4, 5, 6)
)

lapply(werte_liste, mean)
$a
[1] 2

$b
[1] 5
sapply(werte_liste, mean)
a b 
2 5

sapply() bei Data Frames

sapply(daten, mean)
   alter  gewicht     puls 
34.00000 73.66667 75.33333

Das Ergebnis ist hier ein numerischer Vektor.

Numerische Spalten erkennen

patienten <- data.frame(
  id = c("P001", "P002", "P003"),
  alter = c(45, 52, 39),
  gruppe = c("Kontrolle", "Therapie", "Therapie"),
  crp = c(3.2, 7.8, 5.1)
)

sapply(patienten, is.numeric)
    id  alter gruppe    crp 
 FALSE   TRUE  FALSE   TRUE

Mittelwerte numerischer Spalten

numerisch <- sapply(patienten, is.numeric)

sapply(patienten[numerisch], mean)
    alter       crp 
45.333333  5.366667

tapply()

tapply() berechnet Kennwerte getrennt nach Gruppen.

Allgemeine Form:

tapply(X, INDEX, FUN)

Dabei gilt:

  • X: Werte
  • INDEX: Gruppierungsvariable
  • FUN: Funktion

Mittelwert nach Gruppe

crp <- c(3.2, 4.1, 7.8, 6.5, 5.9, 2.8)
gruppe <- c("Kontrolle", "Kontrolle", "Therapie",
            "Therapie", "Therapie", "Kontrolle")

tapply(crp, gruppe, mean)
Kontrolle  Therapie 
 3.366667  6.733333

Weitere Kennwerte nach Gruppe

tapply(crp, gruppe, max)
Kontrolle  Therapie 
      4.1       7.8 
tapply(crp, gruppe, min)
Kontrolle  Therapie 
      2.8       5.9 
tapply(crp, gruppe, median)
Kontrolle  Therapie 
      3.2       6.5

tapply() mit Data Frame

lab <- data.frame(
  patient_id = c("P001", "P002", "P003", "P004", "P005", "P006"),
  gruppe = c("Kontrolle", "Kontrolle", "Therapie",
             "Therapie", "Therapie", "Kontrolle"),
  alter = c(45, 52, 39, 61, 48, 55),
  crp_mg_l = c(3.2, 4.1, 7.8, 6.5, 5.9, 2.8),
  glucose = c(90, 95, 130, 125, 118, 88)
)

tapply(lab$crp_mg_l, lab$gruppe, mean)
Kontrolle  Therapie 
 3.366667  6.733333

Mehrere Kennwerte

tapply(lab$crp_mg_l, lab$gruppe, function(x) {
  c(
    mittelwert = mean(x),
    median = median(x),
    maximum = max(x)
  )
})
$Kontrolle
mittelwert     median    maximum 
  3.366667   3.200000   4.100000 

$Therapie
mittelwert     median    maximum 
  6.733333   6.500000   7.800000

mapply()

mapply() wird verwendet, wenn eine Funktion mehrere Eingaben gleichzeitig braucht.

gewicht <- c(70, 82, 65)
groesse <- c(1.75, 1.82, 1.68)

mapply(function(g, h) g / h^2, gewicht, groesse)
[1] 22.85714 24.75546 23.03005

Typische Fehler bei apply-Funktionen

  • apply() auf gemischte Data Frames anwenden
  • vergessen, dass lapply() eine Liste zurückgibt
  • MARGIN = 1 und MARGIN = 2 verwechseln
  • fehlende Werte nicht beachten
x <- c(3, 4, NA, 6)

mean(x)
[1] NA
mean(x, na.rm = TRUE)
[1] 4.333333

Funktionales Programmieren

Grundidee

Beim funktionalen Programmieren werden Funktionen selbst wie Objekte behandelt.

Das bedeutet:

  • Funktionen können gespeichert werden
  • Funktionen können übergeben werden
  • Funktionen können direkt dort definiert werden, wo sie gebraucht werden

Funktionen sind Objekte

quadrat <- function(x) {
  x^2
}

quadrat(4)
[1] 16

Die Funktion wird im Objekt quadrat gespeichert.

Funktion anzeigen oder ausführen?

quadrat
function (x) 
{
    x^2
}
quadrat(4)
[1] 16

Merksatz:

Ohne Klammern sieht man die Funktion.
Mit Klammern wird die Funktion ausgeführt.

Funktionen speichern

kennzahl <- mean

kennzahl(c(2, 4, 6))
[1] 4

Hier enthält kennzahl die Funktion mean().

Funktionen als Argumente

Eine Funktion kann an eine andere Funktion übergeben werden.

werte <- list(
  a = c(2, 4, 6),
  b = c(10, 20, 30)
)

sapply(werte, mean)
 a  b 
 4 20

mean wird hier an sapply() übergeben.

Anonyme Funktionen

Manchmal braucht man eine Funktion nur einmal.

Dann muss sie keinen eigenen Namen bekommen.

sapply(1:5, function(x) x^2)
[1]  1  4  9 16 25

Die Funktion

function(x) x^2

wird direkt verwendet.

Anonyme Funktion Schritt für Schritt

sapply(1:5, function(x) x^2)
[1]  1  4  9 16 25

Lesart:

  1. Nimm die Werte 1:5.
  2. Wende auf jeden Wert eine Funktion an.
  3. Die Funktion bekommt jeweils einen Wert x.
  4. Für jeden Wert wird x^2 berechnet.

Benannte und anonyme Funktion

quadrat <- function(x) {
  x^2
}

sapply(1:5, quadrat)
[1]  1  4  9 16 25
sapply(1:5, function(x) x^2)
[1]  1  4  9 16 25

Beide Varianten liefern dasselbe Ergebnis.

Wann anonyme Funktionen?

Anonyme Funktionen sind praktisch, wenn die Berechnung:

  • kurz ist
  • nur einmal gebraucht wird
  • direkt im Kontext verständlich ist
werte <- c(1, 10, 100)

sapply(werte, function(x) log10(x))
[1] 0 1 2

Anonyme Funktionen mit Bedingungen

temperatur <- c(36.8, 37.6, 38.4, 39.1)

sapply(temperatur, function(x) {
  if (x < 37.5) {
    "normal"
  } else if (x < 38) {
    "erhöht"
  } else {
    "Fieber"
  }
})
[1] "normal" "erhöht" "Fieber" "Fieber"

Eigene Funktion mit Funktionsargument

analyse <- function(x, FUN) {
  FUN(x)
}

analyse(c(1, 2, 3), mean)
[1] 2
analyse(c(1, 2, 3), sd)
[1] 1

Die Funktion analyse() bleibt gleich.

Nur die übergebene Funktion ändert sich.

Zusätzliche Argumente mit ...

analyse <- function(x, FUN, ...) {
  FUN(x, ...)
}

werte <- c(4, 8, NA, 12)

analyse(werte, mean, na.rm = TRUE)
[1] 8

Die drei Punkte ... geben weitere Argumente weiter.

Funktionen erzeugen Funktionen

Funktionen können auch neue Funktionen zurückgeben.

multiplier <- function(faktor) {
  function(x) {
    x * faktor
  }
}

mal2 <- multiplier(2)
mal5 <- multiplier(5)

mal2(10)
[1] 20
mal5(10)
[1] 50

Prüf-Funktionen erzeugen

groesser_als <- function(grenze) {
  function(x) {
    x > grenze
  }
}

groesser_als_5 <- groesser_als(5)

groesser_als_5(c(3, 6, 9))
[1] FALSE  TRUE  TRUE

Abschluss

Zusammenfassung

Heute haben Sie gelernt:

  • Kontrollstrukturen steuern Programmabläufe
  • Schleifen wiederholen Befehle
  • apply() arbeitet häufig mit Matrizen
  • lapply() gibt immer eine Liste zurück
  • sapply() vereinfacht das Ergebnis
  • tapply() berechnet Kennwerte nach Gruppen
  • anonyme Funktionen sind kurze Funktionen ohne Namen

Mini-Wiederholung

Welche Funktion passt?

Aufgabe Funktion
Mittelwert jeder Matrixspalte apply()
Funktion auf jede Listeneinheit lapply()
kompakter Ergebnisvektor sapply()
Mittelwert nach Gruppe tapply()
mehrere Vektoren parallel mapply()
